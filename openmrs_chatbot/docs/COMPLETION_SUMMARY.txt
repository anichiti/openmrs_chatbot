CORE COMPONENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. TRIAGE AGENT (agents/triage_agent.py)
   Methods:
   • classify_user_type(question) → "DOCTOR" | "PATIENT"
   • classify_intent(question) → Intent category (6 types)
   • extract_patient_id(question) → Patient ID or None
   • triage(question) → Complete triage result

2. SQL AGENT (agents/sql_agent.py)
   Methods:
   • generate_sql_query(question, patient_id) → SQL
   • execute_sql(query) → Results
   • query_patient_record(patient_id) → Full patient data
   • search_patients(name) → Search results
   Safety: SELECT-only, voided filtering, injection prevention

3. MCP AGENT (agents/mcp_agent.py)
   Methods:
   • query_medication_db() → Medication results
   • query_immunization_db() → Vaccine results
   • query_milestone_db() → Developmental milestones
   • search_medication/vaccine/milestone() → Search results
   Data: 5 medications, 5 vaccines, 16 milestone sets

4. KNOWLEDGE AGENT (agents/knowledge_agent.py)
   Methods:
   • query_doctor_kb(question, top_k=5) → Doctor KB results
   • query_patient_kb(question, top_k=5) → Patient KB results
   • format_context(kb_results) → Formatted text
   Backend: ChromaDB with OpenAI embeddings

5. RESPONSE AGENT (agents/response_agent.py)
   Methods:
   • generate_doctor_response() → Clinical response
   • generate_patient_response() → Patient-safe response
   • validate_response_safety() → Safety check
   • format_data_context() → Data formatting
   Output: Structured response with confidence & sources

6. OPENMRS DATABASE (database/db.py)
   Methods:
   • connect/disconnect() → Connection management
   • execute_query() → Safe SQL execution
   • get_patient_by_id() → Patient demographics
   • get_patient_observations() → Recent vitals/labs
   • get_patient_encounters() → Clinical encounters
   • get_patient_conditions() → Diagnoses
   • search_patients() → Patient search

7. VECTORSTORE (vectorstore/chroma.py)
   Methods:
   • index_doctor_kb() → Index doctor PDFs
   • index_patient_kb() → Index patient PDFs
   • query_doctor_kb() → Search doctor KB
   • query_patient_kb() → Search patient KB
   Backend: ChromaDB with persistent storage

8. CLINICAL CHATBOT (main.py)
   Main Class: ClinicalChatbot
   Methods:
   • process_query(question) → Complete response
   • save_response(result) → Persist to responses.json
   • run_interactive() → CLI mode
   Orchestrator: Coordinates all agents
